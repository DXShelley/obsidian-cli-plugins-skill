# Obsidian CLI Plugins 用户使用示例

本文面向使用者，按功能说明可以怎样向 Agent 表达 Obsidian 需求。示例编号 `Uxx` 会在 `agent-usage-examples.md` 中对应到具体脚本调用。

默认目标是当前可解析的 Obsidian vault；需要固定 vault 时，可以在话里说明 `obsidian-2026` 或具体 vault 路径。

## 1. 环境、Vault 和同步

### U01 检查运行环境

```text
检查 Obsidian vault 状态
```

用于新机器、新 Agent、新 shell、OpenClaw 环境或不确定当前 vault 时。

### U02 指定 vault 后检查

```text
使用 vault obsidian-2026，检查 Obsidian 状态
```

适合本机有多个 vault，或 Agent 可能解析到错误 vault 的情况。

### U03 查看 Git 状态

```text
检查 Obsidian vault 的 Git 状态
```

用于确认 vault 是否有未提交、冲突、未完成合并、ahead/behind 等状态。

### U04 同步 vault

```text
同步 Obsidian vault
```

用于拉取、推送和确认同步状态。存在冲突或未完成合并时，Agent 会停止并报告原因。

### U05 使用当前打开的 vault

```text
使用当前打开的 Obsidian vault，查看状态
```

适合 Obsidian 正在打开目标 vault，且不想手写路径时。

## 2. 查看待办

### U06 查看今天新增待办

```text
今日新增待办
```

只查看今天日记中直接新增的任务行，适合快速确认“今天写了哪些新任务”。

### U07 查看今日完整待办

```text
查看今天所有待办，包括逾期和循环任务
```

用于完整 Tasks 查询报告，会包含今日相关、逾期未完成、循环任务等。

### U08 查看本周待办

```text
查看本周待办
```

用于周计划或周回顾，会按 Tasks 查询结果和本周焦点输出。

### U09 查看某周期原始任务行或任务查询结果

```text
查看本月新增任务
```

```text
查看本月任务查询结果
```

前者用于查看月/季/年等周期笔记里直接写入的任务行；后者用于查看该周期的任务查询结果。

## 3. 添加待办

### U10 添加今日待办

```text
添加一个今天的待办：整理 Obsidian 技能用户文档
```

默认会使用同步写入：先做 Git preflight，再写入，再提交和推送。

### U11 添加指定日期待办

```text
添加一个待办：7月5日前完成项目记录功能回归测试
```

Agent 会识别具体日期，并写入合适的周期笔记。

### U12 添加本周、本月、本季、本年任务

```text
把“完善项目记录功能示例”加到本周待办
```

如果没有具体日期，周期任务会使用该周期的默认开始、计划和截止日期。

### U13 本地测试添加待办

```text
本地测试添加一个待办，不提交不推送：验证新增任务章节定位
```

仅用于明确要求本地测试的场景。日常使用建议走同步写入。

## 4. 普通记录

### U14 写入今天记录

```text
记录：今天想到项目需求应该先收集，再逐步细化，不急着马上实现。
```

写入当天 journal 的 `记录` 区域，不创建独立笔记。

### U15 写入周/月/季/年记录

```text
本周记录：项目记录功能需要区分普通记录和项目记录。
```

也可以说 `本月记录`、`本季度记录`、`今年记录`。

### U16 本地测试普通记录

```text
本地测试记录：验证记录章节追加，不提交不推送。
```

仅用于测试写入，不适合作为日常记录方式。

## 5. 文件模式记录和卡片

### U17 创建独立记录卡片

```text
把这条想法创建成单独记录卡片：项目灵感应该有项目文件承接，而不是只放在日记里。
```

适合独立卡片、长内容、需要语义分析、需要日记中放链接的记录。

### U18 记录带来源链接的内容

```text
把这个链接和我的想法记录成一张卡片：https://example.com 这个工具的交互方式值得参考。
```

外部链接会作为来源或原始内容的一部分进入独立记录。

### U19 指定记录类型或主题

```text
创建一张灵感卡片：Obsidian 项目文件可以作为长期项目孵化器。
```

Agent 会把用户原文作为正文，语义分析只用于标题、类型、场景、后续行动等元数据。

## 6. 附件和媒体记录

### U20 同一条消息记录附件

```text
把这张截图记录到 Obsidian：这是项目记录功能的界面草图。
```

只有 Agent 能拿到真实可读的本地附件路径时才会写入；拿不到路径会停止。

### U21 多个媒体文件先发、随后补一句记录

```text
把上面三个媒体文件记录到 Obsidian：项目文件模板设计参考。
```

适用于手机端或 OpenClaw 渠道先发图片、视频、音频或文件，后发文字的场景。Agent 会先暂存媒体文件，再在这句话到来时创建一条带附件记录；这些媒体类型使用同一套处理流程。

### U22 清理过期附件暂存

```text
清理 24 小时以前没有用到的 Obsidian 附件暂存
```

只清理私有缓存中的暂存附件，不删除 vault 里的正式记录附件。

## 7. 项目文件创建

### U23 创建项目文件

```text
创建一个项目：项目记录功能测试，用来验证项目文件创建和小步更新流程。
```

项目文件用于长期积累项目目标、需求、设计、决策、任务和问题，默认放到 `01_project/`。

### U24 指定项目目录和落地判断

```text
在 01_project/obsidian 下创建项目：项目记录功能测试。落地判断是：能稳定创建项目文件，并支持通过 target_id 小步追加内容。
```

适合项目一开始就有明确目录和落地标准的情况。

### U25 本地测试创建项目

```text
本地测试创建项目：项目记录功能测试，不提交不推送。
```

用于验证模板生成、标题和 tail/head 拼接，不作为正式记录流程。

## 8. 项目小步更新

项目小步更新和普通 `记录` 不同：普通记录进 journal，项目记录进具体项目文件。写入前必须读取项目模板结构，并以模板解析结果为准。

### U26 查看项目结构对象

```text
查看项目记录功能测试的项目结构对象
```

用于调试可写入章节、`target_id`、小模板字段和标题层级。

### U27 追加原始灵感

```text
记到项目记录功能测试：想到一个长期项目记录功能，平时一点点补充需求，等清晰后再落地。
```

适合尚未归类的原始想法。

### U28 追加功能需求

```text
项目记录功能测试补充一个功能需求：支持把一句话灵感追加到项目文件对应章节。
```

Agent 会按项目模板和 `card-project-fr` 小模板生成结构化字段。

### U29 追加非功能需求

```text
项目记录功能测试补充约束：项目记录不能依赖硬编码章节，必须以模板结构解析结果为准。
```

Agent 会按 `card-project-nfr` 小模板记录约束、影响、判断标准等。

### U30 追加决策记录

```text
项目记录功能测试记录一个决策：采用 template_structure + target_id 作为写入定位机制。
```

Agent 会按 `card-project-decision` 小模板记录决策、背景、原因和影响。

### U31 追加项目任务或问题

```text
项目记录功能测试加一个任务：补充项目记录功能的用户文档。
```

```text
项目记录功能测试加一个问题：是否需要支持自动识别多个候选项目？
```

任务和问题应写入项目模板 tail 附录中对应位置，避免和正文结构重复。

## 9. 安全读取和搜索

### U32 安全读取 vault 文件

```text
安全读取 01_project/obsidian/项目记录功能测试.md
```

Agent 会使用安全读取规则，避免泄露不必要内容。

### U33 搜索 vault 内容

```text
在 Obsidian 里搜索“项目记录功能”
```

适合定位笔记、标题、片段或项目文件。敏感路径会被跳过或脱敏。

## 10. 插件、命令和官方 CLI

### U34 查看已启用插件

```text
查看 Obsidian 已启用插件
```

用于确认当前 vault 插件环境。

### U35 查找社区插件命令

```text
查找 Obsidian Linter 插件命令
```

适合查找 `obsidian-linter`、`journals`、`obsidian-tasks-plugin` 等社区插件命令。Git 同步优先使用主机 `git`，只有主机 `git` 不存在时才兜底使用 `obsidian-git` 插件命令。

### U36 执行普通插件命令

```text
执行 Obsidian Linter 当前文件格式化命令
```

普通命令可直接执行；敏感或破坏性命令需要明确确认。

### U37 查询官方 Obsidian CLI 命令

```text
查询 Obsidian 官方 CLI 里 property 相关命令
```

用于原生 CLI 命令查找，例如文件、搜索、属性、链接、主题、插件、工作区等。

### U38 执行敏感或破坏性命令

```text
我确认执行 Obsidian 插件禁用命令：plugin:disable
```

只有用户明确确认具体命令和风险后，Agent 才会执行。

## 11. OpenClaw 和跨 Agent 使用

### U39 同步技能到 OpenClaw

```text
把 obsidian-cli-plugins 技能同步到 OpenClaw
```

用于让 OpenClaw 使用同一套脚本、模板解析、附件暂存和记录规则。

### U40 检查 OpenClaw 是否能发现该技能

```text
检查 OpenClaw 里的 obsidian-cli-plugins 是否可用
```

适合同步后验证 OpenClaw 的 skill discovery、脚本路径和 vault 配置。

## 12. 扩展和维护

### U41 为新插件补充工作流支持

```text
给新安装的 Obsidian 插件补充 Agent 工作流支持
```

Agent 会先发现插件、命令和必要配置，再决定是否只更新文档，或把重复逻辑沉淀到 `scripts/`。

### U42 验证技能脚本

```text
验证 obsidian-cli-plugins 技能脚本是否正常
```

适合修改技能后运行测试、`doctor` 和 OpenClaw dry-run。
